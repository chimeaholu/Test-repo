## Delivered Runtime Surface

- Negotiation commands now enforce `open -> pending_confirmation -> accepted/rejected`.
- Final acceptance no longer shortcuts directly from counter to accept.
- Confirmation approval/rejection is restricted to the recorded `required_confirmer_actor_id`.
- Thread reads are actor-membership scoped via:
  - `GET /api/v1/marketplace/negotiations`
  - `GET /api/v1/marketplace/negotiations/{thread_id}`
- Unsupported negotiation mutations such as legacy `market.negotiations.accept` are fail-closed.
- Regulated negotiation mutations emit:
  - command audit events
  - negotiation transition audit events
  - command outbox events
  - negotiation transition outbox events
